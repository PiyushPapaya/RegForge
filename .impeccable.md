# RegForge — Design Context

## Design Context

### Users
Embedded / firmware engineers. Context: at a desk, mid-task, holding a sensor datasheet
PDF and needing a working C driver (register `#define`s, typed read/write helpers, a cited
power-on init sequence). They are skeptical of LLM output by default — their job is to
**verify** an extracted register map before trusting generated code. Used on desktop
primarily; must stay usable on a phone.

### Brand Personality
Precise · instrument-grade · trustworthy. The product should feel like a piece of bench
test equipment or an oscilloscope UI — calm, exact, legible, never flashy. Confidence comes
from restraint and from showing its work (every output cites a datasheet page).

### Aesthetic Direction
Refined dark "lab instrument." Near-black surfaces (`#08090b`), a **reserved** phosphor-green
accent used only for primary action and the live/active state, steel-blue strictly for
references/citations, amber for low-confidence warnings. Theme is dark by deliberate choice:
the audience works in editors/terminals and the citation→source pairing reads best on dark.
Type: Geist Sans (body/UI) + Geist Mono (data, code, labels) with a strong size/weight
hierarchy — mono is used for *real* technical content (addresses, code, stage labels), never
as decorative "techy" shorthand. Anti-references: generic SaaS gradient hero, neon
glassmorphism, purple→blue AI gradients, cyan-on-black.

### Design Principles
1. **Accent is rare.** Phosphor green = primary action + "happening now" only. If everything
   glows, nothing does. Steel-blue = citations. Calm neutrals carry the rest (60-30-10 by weight).
2. **Show the work.** Trust is earned by visible verification and page citations, not copy.
   The register table and the source pane are the heart of the product.
3. **Refine, don't reinvent.** Evolve the established instrument aesthetic; no gradient text,
   no decorative blur, no AI-tell flourishes.
4. **Legible density.** Technical content can be dense, but never cramped — rhythm via spacing
   scale, ≥12px for primary text, generous separation between groups.
5. **Motion is meaning.** Animate state changes (extraction streaming, code assembly, stage
   handoff) with smooth ease-out; everything respects `prefers-reduced-motion`.
